## ✨ 特性

- **三种比对模式**：
  - 🔍 **精确模式**：适用于原始无水印视频，速度快
  - 🛡️ **抗水印模式**：适用于带字幕/水印的视频
  - ⚡ **高精度模式**：五维特征融合，准确率最高

- **智能缓存系统**：自动缓存扫描结果，二次扫描速度提升10倍以上
- **多线程加速**：支持并发处理，充分利用CPU资源
- **可视化预览**：实时显示重复视频组和相似度评分
- **一键选择**：智能推荐保留最优质量的视频
- **国际化支持**：支持中文/英文界面切换
- **安全删除**：文件移至回收站，可随时恢复

## 📖 使用说明

### 1. 选择视频目录
点击"浏览"按钮选择包含视频的文件夹，可选择是否包含子文件夹。

### 2. 选择比对模式
- **精确模式**：适合原始视频，使用pHash算法，3帧采样
- **抗水印模式**：适合有字幕/水印的视频，6帧采样，裁剪中心区域
- **高精度模式**：最准确但较慢，5种特征融合，10帧采样

### 3. 调整参数
- **线程数**：根据CPU核心数调整（建议4-8线程）
- **相似度阈值**：数值越低要求越严格

### 4. 开始扫描
点击"▶ 开始扫描"按钮，等待扫描完成。

### 5. 查看结果
右侧面板显示所有重复视频组，包括：
- 相似度评分（绿色<5，橙色5-12，红色>12）
- 文件大小、分辨率、时长等信息
- 预览缩略图

### 6. 选择要删除的文件
- **手动选择**：勾选要删除的视频
- **一键选择**：自动选择最小文件或最低分辨率的视频
- **筛选功能**：按相似度范围筛选显示

### 7. 删除文件
点击"🗑️ 删除选中文件到回收站"，文件将被移至回收站而非永久删除。

## ⚙️ 设置选项

点击"⚙️ 设置"按钮可配置：

- **基础选项**：
  - 多线程加速
  - 自动旋转竖屏视频
  - 统一分辨率比较

- **预览设置**：自定义缩略图尺寸

- **进度管理**：导出/导入扫描进度备份

- **缓存管理**：清空不同模式的缓存

- **语言设置**：切换中文/英文界面

## 💡 最佳实践

### 性能优化
1. **首次扫描**：建议使用4-8线程，平衡速度和内存
2. **高精度模式**：限制为2-4线程，避免内存溢出
3. **大文件库**：启用缓存机制，二次扫描极快

### 准确度调优
1. **严格模式**：降低相似度阈值（精确模式：5-8，高精度：3-5）
2. **宽松模式**：提高阈值以捕获更多相似视频
3. **抗水印**：对于有硬字幕的视频必须使用抗水印模式

### 数据安全
1. **定期备份**：使用"导出进度备份"保存扫描进度
2. **谨慎删除**：删除前仔细预览，确认是重复文件
3. **回收站恢复**：误删文件可从回收站恢复

## 📝 技术细节

### 比对算法

#### 精确模式
- 单特征：pHash感知哈希
- 采样：3帧（开头、中间、结尾）
- 判定：所有帧哈希距离均≤阈值

#### 抗水印模式
- 单特征：pHash（裁剪中心区域+模糊处理）
- 采样：6帧均匀分布
- 判定：平均汉明距离≤阈值

#### 高精度模式
- 五维特征融合：
  - pHash（感知哈希）
  - aHash（平均哈希）
  - dHash（差异哈希）
  - 颜色直方图
  - SSIM（结构相似度）
- 采样：10帧均匀分布
- 加权评分：综合计算相似度

### 缓存机制
- **分片存储**：将缓存分为多个小文件，避免单文件过大
- **增量更新**：仅保存变化的数据
- **按需加载**：只加载当前扫描目录的缓存
- **自动验证**：通过文件大小和修改时间验证缓存有效性

### 代码结构
- `main.py`：App类继承自tk.Tk，负责整体流程控制
- `ui.py`：create_ui函数构建界面，包含国际化支持
- `core.py`：三种模式的核心算法实现
- `scanner.py`：后台扫描线程，支持暂停/停止
- `video_cache.py`：缓存读写和管理

### 添加新功能
1. 在`config.py`中添加配置参数
2. 在`core.py`中实现核心逻辑
3. 在`ui.py`中添加界面控件
4. 在`main.py`中绑定事件处理
